# DOMAIN MODEL v2
## USER
<b>DESCRIPTION</b>: THIS TABLE IS USED TO RECORD THE TYPES OF USERS WHO HAVE ACCESS TO THE SYSTEM. THIS SHALL STORE THEIR INFO AND THEIR PASSWORD HASHCODES, WITH OTHER NEEDED INFORMATION.
 
- USER_ID
- USERNAME
- EMAIL
- PASSWORD_HASH
- ROLE
- IS_ACTIVE
- LAST_LOGIN
- CREATED_AT
- UPDATED_AT

## PRODUCT
<b>DESCRIPTION</b>: THIS TABLE IS USED TO RECORD THE PRODUCT’S INFO, ITS DESCRIPTION, WHO IT HAS BEEN SUPPLIED FOR, CURRENT INVENTORY COUNT, CATEGORY AND OTHER INFORMATION LIKE BARCODE AND SKU, ALONG WITH REORDER LEVEL. REORDER LEVEL WILL BE USED TO KEEP A CHECK ON THE COUNT OF THE ITEMS AVAILABLE IN STOCK, AND IN CASE THE COUNT DROPS, A NOTIFICATION MESSAGE CAN BE SENT OUT OR A POP SCREEN CAN HELP. 
- PRODUCT_ID
- PRODUCT_NAME
- BARCODE
- SKU
- DESCRIPTION
- CATEGORY_ID
- SELLING_PRICE
- QUANTITY
- REORDER_LEVEL 
[	Example:
Current Stock = 10
Reorder Level = 15
	Dashboard shows: LOW STOCK ALERT 	]
- IS_ACTIVE
- CREATED_AT
- UPDATED_AT

## CATEGORY
<b>DESCRIPTION</b>: THIS TABLE IS USED TO RECORD THE CATEGORIES IN WHICH THE PRODUCTS FALL IN 
- CATEGORY_ID
- CATEGORY_NAME
- DESCRIPTION

## CUSTOMER
<b>DESCRIPTION</b>: THIS TABLE IS USED TO RECORD THE CUSTOMER DATA. THEIR INFO, ANY OWED BALANCES AND STATEMENTS. NOTE: A SALE CAN OCCUR WITHOUT A CUSTOMER AS WELL, CONSIDERING THAT AS A GUEST CUSTOMER. 
- CUSTOMER_ID
- CUSTOMER_NAME
- PHONE
- EMAIL
- ADDRESS
- BALANCE
- CREATED_AT
- UPDATED_AT

## SUPPLIER
<b>DESCRIPTION</b>: THIS TABLE IS USED TO RECORD THE SUPPLIER DATA. THEIR CONTACT INFO AND WHAT PRODUCTS DOES THE BUSINESS RECEIVE FROM THEM. AND ALSO THE BALANCE OWED OR OWING HISTORY.
- SUPPLIER_ID
- SUPPLIER_NAME
- CONTACT_PERSON
- PHONE
- EMAIL
- ADDRESS
- BALANCE
- CREATED_AT
- UPDATED_AT

## SALE
<b>DESCRIPTION</b>: THIS TABLE IS USED TO RECORD THE SALE, WHICH INCLUDES MULTIPLE SALE ITEMS, AND THE CUSTOMER ID IS OPTIONAL AS IT CAN BE A GUEST CUSOMTER AS WELL. THIS TABLE HAS OTHER SALE ASSOCIATED DETAILS ATTAHCED TO IT. 
- SALE_ID
- INVOICE_NUMBER
- CUSTOMER_ID
- DATE
- TOTAL_AMMOUNT
- DISCOUNT
- TAX
- PAYMENT_STATUS [ VALUES: PAID, PARTIAL, UNPAID ]
- PAYMENT_METHOD [ VALUES: CASH, UPI, CARD, BANK_TRANSFER ]
- CREATED_BY

## SALE-ITEM
<b>DESCRIPTION</b>: THIS TABLE IS USED TO RECORD THE SALE ITEMS INSIDE A SALE. THIS REFLECTS ON THE QUANTITY AND OTHER DETAILS AND IS ASSOCIATED WITH THE SALE TABLE 
- SALE_ITEM_ID
- SALE_ID
- PRODUCT_ID
- QUANTITY 
- SELLING_PRICE
- SUBTOTAL

## PRODUCT_SUPPLIER
<b>DESCRIPTION</b>: THIS TABLE IS A JUNCTION TABLE AS A RESULT OF N:N RELATIONSHIP BETWEEN THE PRODUCT AND THE SUPPLIER TABLE  
- PRODUCT_SUPPLIER_ID
- PRODUCT_ID
- SUPPLIER_ID
- COST_PRICE
- LAST_ORDERED_DATE
- CREATED_AT
- UPDATED_AT

## THIS EFFECTIVELY CHANGES THE RELATIONSHIPS TO AS BELOW : 
````
CATEGORY
   |
   | 1:N
   |
PRODUCT – – 1:N – – PRODUCT_SUPPLIER – – N:1 – – SUPPLIER
   |
   | 1:N
   |
SALE_ITEM
   |
   | N:1
   |
SALE 
   |
   | N:1
   |
CUSTOMER
``````
